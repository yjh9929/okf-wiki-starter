오늘의 목표

4시간 뒤, 여러분은
나만의 지식 위키 하나를 갖게 됩니다

관심 자료를 넣으면 AI가 알아서 정리·연결하고, 질문에 답하고, 스스로 점검하는 — 평생 쓰는 개인 지식 창고.

왜 이렇게 하나

"물어볼 때마다 처음부터 다시 뒤진다"

요즘 흔한 AI 검색 방식(RAG)은 질문할 때마다 원본 문서를 매번 다시 읽습니다.

→ 같은 자료를 반복해 읽어 비용이 계속 들고, 조각조각 읽다 보니 내용이 어긋나거나 틀린 답이 나오기 쉽습니다.

발상의 전환

모을 때 미리 정리해 두면 어떨까?

Andrej Karpathy의 LLM Wiki 아이디어 — 자료를 처음 모을 때 AI가 요약·연결해 위키로 만들어 둡니다. 이후엔 잘 정리된 요약만 보면 되니 싸고 일관됩니다.

"Obsidian은 작업실, AI는 정리해 주는 사서, 위키는 차곡차곡 쌓이는 내 서재."

질문할 때마다 조립하지 말고,
모을 때 미리 컴파일하라.

지식의 표준 형식

OKF — 어렵지 않습니다

구글의 Open Knowledge Format. 그냥 마크다운 파일에 짧은 표식 하나(type)만 붙이면 끝.

개념 하나 = 파일 하나. Obsidian에서 열면 개념들이 실처럼 연결된 지식 그래프로 보입니다. 사람도 기계도 함께 읽을 수 있는 최소 표준.

Part 1

시작하기 — 환경 세팅

가장 중요한 건 세팅. 한 번만 잘 해두면 수업 후에도 계속 씁니다.

환경 세팅

6단계면 준비 끝

  1. Claude Code 설치·로그인
  2. 실습 repo 내려받기 (말로 시키기)
  3. Obsidian 설치·열기
  4. 웹 클리퍼 확장 [필수]
  5. 커뮤니티 플러그인 3개 [추천]
  6. 확인 → 첫 실습!

Step 2 · 내려받기

git을 몰라도 됩니다

Claude Code에게 말로 부탁하세요:

"이 repo를 my-wiki 폴더로 clone 해줘: github.com/kisdevan/okf-wiki-starter"

스킬·예제·점검 도구가 모두 담긴 폴더가 생깁니다. (Windows는 Git for Windows만 미리 설치)

준비물

딱 이것만 있으면 됩니다

  • Claude Code · Obsidian · 본인 Claude 구독
  • Windows라면 Git 하나 더
  • Node.js는 필요 없어요 — 심화(llmwiki)에서만

Step 4·5 · 확장과 플러그인

수업 후에도 편하게 쓰려면

✂️
웹 클리퍼 [필수]
공식 브라우저 확장. 좋은 웹 글을 raw/에 바로 저장. 보관함=my-wiki, 저장위치=raw로 설정.
🧠
플러그인 3개 [추천]
Dataview(type별 모아보기) · Templater(양식 자동) · Omnisearch(검색). Obsidian 안에서 설치.

그래프 뷰와 속성(Properties)은 기본 내장 — 따로 설치할 게 없어요.

Part 2

실습 — 정리 · 질문 · 점검

이제 진짜 위키를 만들어 봅니다. 명령은 전부 자연어로.

모듈 ③ · Ingest

자료를 넣으면, 위키가 됩니다

이렇게 말하세요:

"raw 폴더 읽어서 위키로 정리해줘"

→ wiki/에 개념 문서와 목차(index)가 생기고, 겹치는 내용은 하나로 합쳐지며, 각 사실에 출처가 붙습니다. Obsidian 그래프에 점들이 이어져요.

모듈 ③ · Query

내 위키에 물어보기

"위키 내용 근거로 'OKF가 뭐야?' 알려줘. 출처도 달아줘. 쓸만하면 저장해."

출처가 달린 답이 나오고, 좋은 답은 다시 위키에 쌓입니다.

쓸수록 똑똑해지는 것 — 이게 "복리처럼 자라는 지식"입니다.

Live Demo

함께 /okf-wiki 실행해 봐요

raw 예제 2개 → 위키 만들기 → 질문하기

모듈 ④ · Lint

그래프로 보고, 자동 점검

Obsidian 그래프에서 외톨이 노드(연결 안 된 문서)를 찾아 채우고, 점검을 돌립니다.

"위키 점검해줘"

→ 끊긴 링크·빈 표식·외톨이 문서를 자동으로 찾아 줍니다. 통과하면 완성!

지금 바로 · 30초

첫 실습, 이 순서대로

  1. 정리 — "raw 폴더 읽어서 위키로 정리해줘"
  2. 질문 — "위키 근거로 OKF가 뭐야? 출처도"
  3. 점검 — "위키 점검해줘"

오늘의 결과물

손에 남는 것

  • 목차(index)와 개념 문서 3~5개
  • 질문에 답한 노트 (Query 결과)
  • 연결이 살아 있는 지식 그래프
  • 점검 통과 — 끊긴 링크 0, 외톨이 0

계속 쓰기

수업이 끝나도, 여기서부터 시작

✂️
모으기
좋은 웹 글은 웹 클리퍼로 raw/에 저장 → 나중에 위키로 편입.
💬
물어보기
공부하다 막히면 "내 위키 근거로 설명해줘".
🔁
다듬기
가끔 "위키 점검해줘"로 정리 — 쓸수록 똑똑해집니다.
📦
어디서든
그냥 마크다운 폴더라, 다른 컴퓨터·도구로 옮겨도 그대로.
