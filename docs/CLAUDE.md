# 강의 사이트 작업 규칙

이 파일은 Claude Code가 이 폴더에서 작업할 때 먼저 읽는 문서다.
새 화면을 만들거나 기존 화면을 고칠 때 **아래 규칙을 그대로 따른다.**

---

## 1. 이게 무엇인가

연세대학교 × 모두의연구소 4시간 세미나 **"나만의 AI 지식 관리 시스템 만들기"** 의
타임테이블 겸 강의자료 사이트다. GitHub Pages로 배포한다.

- **듣는 사람**: 연세대 학부생 20명. **코딩 경험이 없다고 가정한다.**
- **쓰는 상황**: 강사가 프로젝터에 띄워 PPT 대신 넘기고, 학생은 나중에 혼자 스크롤해 읽는다.
- **핵심 메시지**: 질문할 때마다 다시 뒤지지 말고, 모을 때 미리 정리해 두자 (RAG → LLM Wiki)

```
docs/
├── index.html      타임테이블 (시작 페이지)
├── m0.html         0-1 오프닝
├── m1.html         1-1 ① 지식관리 에이전트를 위한 LLM wiki와 OKF 이해하기
├── m2.html         2-1 ② 클로드 환경 세팅
├── m3.html         3-1 ③ 내 문서로 지식 관리 에이전트 시작하기 — 정리
├── m4.html         3-2 ③ 내 문서로 지식 관리 에이전트 시작하기 — 질문
├── m5.html         4-1 ④ 연결망으로 보는 나만의 지식 위키 완성하기
├── m6.html         5-1 발표하기
├── m7.html         0-2 마무리
└── assets/
    ├── style.css        전체 스타일 (색·글꼴이 전부 여기 :root에 있다)
    └── deck.js          방향키 넘기기, 진행 막대
```

각 HTML은 **독립 파일**이다. 공통 부분(상단 바, head)은 파일마다 복사돼 있으니,
공통 요소를 바꿀 때는 **9개 파일을 모두** 고쳐야 한다.

## 2. 디자인 토큰

`assets/style.css` 맨 위 `:root` 에 전부 정의돼 있다. **색과 글꼴은 여기서만 바꾼다.**
HTML 안에 `style="color:#..."` 같은 인라인 색상을 새로 만들지 않는다.

| 변수 | 값 | 쓰는 곳 |
|---|---|---|
| `--green` | `#0F4B43` | 제목, 표지 배경, 코드 블록 배경, 기본 버튼 |
| `--leaf` | `#45A216` | 강조, 링크 밑줄, 연결선, 번호 동그라미 |
| `--leaf-dark` | `#37800F` | 링크 글자 (본문 대비 확보용) |
| `--leaf-soft` | `#EAF3E4` | 강조 카드 배경, 짝토론 상자 |
| `--paper` | `#EFE7E1` | 페이지 배경 |
| `--surface` | `#FBF8F5` | 카드·표·상자 배경 |
| `--taupe` | `#D5C6B9` | 테두리 |
| `--grey` | `#BEB4B4` | 도형 선, 체크박스 |
| `--slate` | `#565855` | 본문 글자 |
| `--muted` | `#8E8781` | 라벨, 캡션 |

**색을 쓰는 원칙 하나**: 초록(`--leaf`)은 **"이어지는 것"에만** 쓴다.
링크, 연결선, 강조 표시, 새로 추가된 줄. 장식으로 쓰지 않는다.

## 3. 글꼴

```css
--sans:    Pretendard          /* 본문 */
--display: IBM Plex Sans KR    /* 제목 (h1~h4) */
--mono:    IBM Plex Mono       /* 시각, 경로, 라벨, 코드 */
```

- 세 글꼴 모두 **CDN에서 받아온다.** Pretendard는 jsdelivr, IBM Plex는 Google Fonts.
- 인터넷이 없으면 시스템 기본 글꼴로 대체된다. 레이아웃은 유지되지만 인상이 달라진다.
- **글꼴을 새로 추가하지 않는다.** 이 세 개로 충분하다.
- 굵기는 500 / 600 / 700만 쓴다. 그 외 굵기는 CDN에서 받아오지 않으므로 가짜 볼드가 된다.

## 4. 한 장(슬라이드)의 구조

```html
<section class="slide"><div class="wrap">
  <div class="eyebrow">작은 라벨</div>
  <h2>큰 제목</h2>
  <p class="lead">한두 문장 설명</p>
  <!-- 본문 요소 -->
</div></section>
```

**표지는 `class="slide cover"`** — 딥그린 배경, 흰 글씨.

```html
<section class="slide cover"><div class="wrap">
  <div class="num">3-1 · 모듈 ③ · 30분</div>
  <h1>두 줄까지<br>넘어가는 제목</h1>
  <div class="meta"><span class="tag">키워드</span><span class="tag">키워드</span></div>
</div></section>
```

## 5. 쓸 수 있는 조각들

**이미 있는 것만 쓴다. 새 CSS 클래스를 만들지 않는다.**

| 클래스 | 언제 쓰나 |
|---|---|
| `.grid.g2` `.g3` `.g4` + `.card` | 개념 2~4개를 나란히 놓을 때 |
| `.card.accent` | 그중 하나를 강조할 때 (초록 배경) |
| `.steps` (ol) | 순서가 있는 단계. 자동으로 번호가 붙는다 |
| `.check` (ul) | 확인 목록. 체크박스 모양이 붙는다 |
| `table.t` | 항목 비교, 증상-해결 대응표 |
| `.vs` + `.side` / `.side.b` / `.mid` | 두 방식을 맞세울 때 |
| `.say` | 학생이 실제로 입력할 문장 (`<span class="k">라벨</span><q>문장</q>`) |
| `.talk` | 짝토론 안내 (초록 상자) |
| `.note` | 보충 설명, 주의사항 (점선 테두리) |
| `.tally` | 큰 숫자 나열 (0, 0, 0, 0) |
| `.spec` + `.box` | 커리큘럼 원문을 보여 주는 장 |
| `.topics` | 예시 주제 목록 |
| `.bio` | 강사 소개 (사진 없음) |
| `pre.code` | 명령어·파일 예시 |
| `.path` | 파일 경로나 명령어를 문장 안에 넣을 때 |
| `.wl` | `[[wiki/...]]` 링크 표기 (대괄호는 자동으로 붙는다) |
| `.mark` | 형광펜 강조. **한 장에 한 번만** |
| `figure.dia` + `ol.shots` | 화면 예시 그림과 번호 설명 |

## 6. 반드시 지킬 것

0. **모듈 표지(`.cover`)의 제목은 커리큘럼 엑셀의 모듈명 그대로 쓴다.**
   더 그럴듯한 제목을 새로 지어내지 않는다. 표지의 태그도 엑셀 `핵심 내용` 한 개만 넣는다.

1. **한 장은 한 화면에 들어와야 한다.**
   1280×720(프로젝터 기준)에서 스크롤 없이 보여야 한다. 내용이 넘치면 **장을 나눈다.**
   확인: 브라우저 창을 1280×720으로 줄여 본다.

2. **화면 예시 그림은 `max-height:42vh`** 로 제한돼 있다. 이 값을 올리지 않는다.

3. **한 장에 한 가지 이야기만.** 제목이 두 가지를 말하고 있으면 나눈다.

4. **이론이 연속 20분을 넘지 않게** 중간에 `.talk`(짝토론)나 실습 장을 끼운다.

5. **내부 링크는 상대 경로**로만 쓴다. (`m3.html`, `assets/style.css`)

6. **브라우저 저장소(localStorage 등)를 쓰지 않는다.** 정적 페이지로만 동작해야 한다.

7. **이모지를 쓰지 않는다.**

## 7. 글을 쓸 때

**말투**

- **설명하듯 말고 선언하듯 쓴다.** 제목은 그 장에서 할 일이나 결과를 그대로 적는다.
  "여기까지 됐나요" → "여기까지 확인합니다" / "두 가지면 충분합니다" → "두 가지를 준비합니다"
- **한 문단은 두 문장까지.** 부연, 여담, "~인 셈이니까요" 같은 꼬리 문장을 붙이지 않는다.
- **존댓말로 통일한다. 반말 종결(`-ㄴ다`, `-는다`)을 섞지 않는다.**
  "말로 시킨다" → "한국어로 요청하기" / "물어볼 때마다 찾는다" → "물어볼 때마다 찾습니다"
- **자리마다 형식이 정해져 있다.**
  - `h2`(장 제목) → 존댓말 문장. "정리를 실행합니다"
  - `.card h4` / `.side h4` → **명사구 또는 `-기` 형**. "결과 확인하기", "중심 개념"
  - `.steps li b`(단계 제목) → 존댓말 문장. "찾기 쉬운 곳에 압축을 풉니다"
  한 장 안에서 이 형식을 섞지 않는다.
- 구어체 단어를 쓰지 않는다: `그냥`, `잔뜩`, `짜다`(코드를 짜다 → 작성하다), `갈아 끼우다`, `~인 셈이다`
- 학생에게 말하듯 담백하게. 과장·감탄사·이모지 없음.
- 번역투 금지: `~을 통해`, `~에 대한`, `~되어지다`, `지속적으로`, `~하는 것을 확인할 수 있습니다`
- 명사 나열 대신 동사로: "출처 표기 방식 이해" → "출처가 왜 달리는지 알게 됩니다"
- 전문 용어를 굳이 쓰지 않는다: `고아 노드(Orphan)` → `외톨이 문서`, `피어 시연` → `발표`

**금지**

- **작업 지시문을 화면에 노출하지 않는다.** "커리큘럼에 적힌 그대로", "이 시간에 하는 일",
  "아래 내용을 참고하여" 같은 문장은 학생에게 보여 줄 말이 아니다.
- 학생이 못 할 일을 시키지 않는다. 이 수업에서 **터미널·명령어·Node.js는 쓰지 않는다.**

**커리큘럼 원문**

각 모듈 두 번째 장(`.spec`)의 문장은 **커리큘럼 엑셀 원문 그대로**다.
줄바꿈, `- ` 머리표, `→` 화살표, 빈 줄, 띄어쓰기까지 손대지 않는다.
엑셀이 바뀌면 **`index.html`의 해당 행과 그 모듈의 두 번째 장, 두 곳을 함께** 고친다.

## 8. 사실관계 (틀리면 안 되는 것)

- 세미나 날짜: **2026년 8월 26일 수요일**
- 실습 저장소: `https://github.com/yjh9929/okf-wiki-starter`
- 환경 세팅은 **Claude 데스크톱 앱의 Code 탭** 기준. 터미널과 Node.js는 쓰지 않는다.
- **윈도우만 Git 설치가 필요하다.** 로컬 폴더를 다루는 데 쓰이며, 학생이 git 명령을 칠 일은 없다.
- OKF에서 **필수 항목은 `type` 하나뿐이다.**
- 위키 구조: `raw/`(원본, 수정 금지) · `wiki/`(AI가 생성) · `wiki/index.md`(목차) · `wiki/log.md`(기록)
- 세 가지 작업: 정리(Ingest) · 질문(Query) · 점검(Lint)
- 점검 항목 네 가지: 끊긴 링크 · 고립된 노트 · 표식 누락 · 출처 누락

## 9. 자주 하는 작업

### 장 하나 추가하기
해당 `mN.html`을 열고, 원하는 위치에 `<section class="slide">` 블록을 넣는다.
상단 바의 장 번호(`1 / 13`)는 `deck.js`가 자동으로 센다. 손대지 않는다.

### 색 바꾸기
`assets/style.css` 의 `:root` 변수만 고친다. 다른 곳은 건드리지 않는다.

### 화면 예시 그림 고치기
각 HTML 안에 인라인 SVG로 들어 있다. `viewBox`는 그대로 두고 내부 좌표만 조정한다.
실제 화면 캡처로 바꾸려면 `<svg>...</svg>` 자리에 `<img src="assets/파일명.png" alt="설명">` 을 넣는다.

### 화면 예시 번호 배지
`ol.shots li` 안의 번호는 반드시 `<span class="n">1</span>` 로 쓴다.
`<b>` 로 쓰면 본문의 굵은 글씨까지 배지로 잡혀 글자가 세로로 깨진다.

### 배포
`docs/` 폴더를 저장소에 올리고, Settings → Pages → Branch: `main` / `/docs`.

## 10. 고칠 때 쓸 프롬프트 예시

그대로 복사해 쓰면 된다.

```
m2.html 의 5단계 장에 "저장 위치를 못 찾는 경우" 항목을 note 로 하나 추가해 줘.
CLAUDE.md 의 규칙을 지키고, 1280x720 에서 한 화면에 들어오는지 확인해 줘.
```

```
m1.html 의 RAG 설명 장이 너무 길어. 두 장으로 나눠 줘.
기존 클래스만 쓰고 새 CSS 는 만들지 마.
```

```
index.html 타임테이블의 3-1 행 문구를 아래 엑셀 원문으로 교체해 줘.
줄바꿈과 화살표를 그대로 유지하고, m3.html 두 번째 장도 같이 맞춰 줘.
(원문 붙여넣기)
```

```
전체 파일에서 "지속적으로" 같은 번역투 표현을 찾아서 자연스러운 한국어로 고쳐 줘.
바꾸기 전에 목록으로 먼저 보여 줘.
```

```
모든 mN.html 의 상단 바에 있는 안내 문구를 "← → 넘기기 · F 전체화면" 으로 통일해 줘.
9개 파일 전부 확인해 줘.
```
